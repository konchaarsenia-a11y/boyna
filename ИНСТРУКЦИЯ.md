## Часть 1. Обновить код в Google (обязательно первым!)

Без этого мини-апп будет стучаться в **старый** сервер — позиции снова не запишутся.

### Шаг 1 — Открыть редактор скрипта

1. Откройте таблицу:  
   https://docs.google.com/spreadsheets/d/1aBNcgobp5GNBKySjMKRWEDWWKebF5kqb5A-cZoDuvG8/edit  
2. Меню сверху: **Расширения** → **Apps Script**.  
3. Откроется новая вкладка с кодом.

### Шаг 2 — Заменить весь код

1. В левой панели кликните файл (обычно `Code.gs`).  
2. **Выделите весь текст** (Ctrl+A) и **удалите**.  
3. Откройте на компьютере файл из проекта:  
   `F:\всякое\Git\projects\superboyna\Code.gs`  
4. Скопируйте **весь** файл и вставьте в Apps Script.  
5. Нажмите **Сохранить** (иконка дискеты или Ctrl+S).

### Шаг 3 — Опубликовать новую версию (Deploy)

1. Справа сверху кнопка **Развернуть** (Deploy).  
2. **Управление развёртываниями** (Manage deployments).  
3. У вашего веб-приложения нажмите карандаш **✏️ Изменить**.  
4. В поле **Версия** выберите **Новая версия**.  
5. Нажмите **Развернуть**.  
6. Если спросит доступ — выберите свой Google-аккаунт → **Разрешить**.

Готово. URL `/exec` обычно **тот же** — в `app.html` уже прописан текущий.

Если Google показал **новый** URL — скопируйте его и замените строку `GOOGLE_WEBHOOK_URL` в `app.html`.

1. Откройте файл:  
   `F:\всякое\Git\projects\superboyna\app.html`

4. В Telegram: закройте мини-апп полностью и откройте снова (чтобы подтянулся новый файл, не из кэша).

Если не помните, куда выкладывали HTML — напишите мне, разберёмся вместе.

## Часть 3. Как проверить, что всё работает

### Менеджер → Заказ
1. Вкладка **Заказ**.  
2. Клиент: `zzz_test`, день например **Вторник**.  
3. Добавьте 1–2 позиции (например ЛЁГКОЕ / Среднее 111 гр + УХО Г 2 шт).  
4. Адрес и примечание — любые.  
5. **Сохранить заказ**.  
6. Должно написать, сколько позиций сохранилось.  
7. В таблице на «Прием заказов» во **вторнике** появится столбец `zzz_test` с цифрами.

### Менеджер → Неделя
1. Вкладка **Неделя** → тот же день.  
2. Видны клиенты, **гр** и **шт** правильно.  
3. Кнопки: ✏️ редактировать, 🔄 перенос, 🗑️ удалить — проверить на `zzz_test`.

### Нарезка
1. Вкладка **Нарезка** → день с заказами (например Понедельник).  
2. Список только нужных позиций: сухое + сырое.  
3. Галочка «Нарезано», поле «Излишек» → сохранить.

### Курьер
1. Вкладка **Курьер** → день.  
2. Клиенты, составы, адреса.  
3. Галочки «доставлен».  
4. Кнопка **Яндекс.Карты** — откроет маршрут по адресам (у кого адрес заполнен).

### Таблица: завершить неделю
После деплоя `Code.gs` при закрытии недели адреса и примечания с «Будущей недели» переносятся на «Прием заказов» (строки 60–61).  
**Не нажимайте** «Завершить неделю» для теста без необходимости — это сдвигает всю неделю.

## Если что-то не работает

| Симптом | Что сделать |
|---------|-------------|
| Позиции не пишутся | Не сделан Deploy новой версии Code.gs |
| «Ошибка сети» / вечная загрузка | Проверьте интернет; webhook URL в app.html |
| Нарезка/курьер «unknown_action» | Старый Deploy — снова Часть 1, шаг 3 |
| Яндекс.Карты пустые | Нет адресов у клиентов в таблице |
