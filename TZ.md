## C. Баги и UX (починить в общей пачке)

### C1. Instagram-чеклист
- [ ] Плохо вносит позиции — починить парсер (фракции, объёмы, имена, рубец/корень и т.д.)
- [ ] Во вкладке **Цена** — тот же чеклист, что в подписке/заказе (не «из корзины в корзину»)

### C2. Примечания в Заказе
- [ ] Сейчас роли на одном тексте; нужно: **несколько примечаний** — кнопка **«Добавить ещё»** (например одно курьеру, одно нарезчику)
- [ ] При сохранении спрашивать: примечание **постоянное** или **на один раз**; писать в данные с пометкой; постоянное тянуть в следующие заказы клиента; разовое — только этот заказ

### C3. Нарезка
- [ ] Галочки снова сохраняются криво — добить надёжное сохранение/опрос
- [ ] **Убрать анимацию loading-dance (тянка)** — простой индикатор загрузки

### C4. Курьер / Сборка
- [ ] **Сборка** открывается **long-press** (как подменю), не отдельным табом на виду постоянно (или как согласуем — см. вопросы)
- [ ] Счётчики пакетов: показывать **только тип пакета + количество** (без лишнего)
- [ ] ПП N=2: возле «доставлено» всегда вопрос «оплачено ли?»

### C5. Фракции
- [ ] **Корень / рубец** (уточнить позицию) — фракция не отображается нигде в приложении — починить отображение везде (корзина, просмотр, нарезка, курьер, сборка)

### C6. Скорость
- [ ] Приложение долго грузит из таблицы — ускорить (кэш, меньше round-trip, параллельные запросы, урезать payload)

### C7. Цена
- [ ] Режим ПП / БП / Розница → сразу считать стоимость; для розницы — по прайсу (после уточнения позиций)
