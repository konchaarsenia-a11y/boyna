# ТЗ — Telegram Mini App «Бойня-Конвейер»

Обновлено: 2026-07-19. Источник: владелец (Арсений).

## Отложено (напомнить владельцу)

- **Перевыпуск Telegram bot token** — токен светился в чате; сделать позже через @BotFather → Revoke, положить в Script Properties.

## Приоритет работ

1. **Менеджер — приём заказа** — починить запись позиций (имя/адрес/примечание пишутся, позиции — нет).
2. **Менеджер — просмотр недели** — гр/шт, кнопки редактировать / удалить / перенести.
3. **Нарезчик** — новый экран.
4. **Курьер** — новый экран.
5. Таблица: «Завершить неделю» — перенос адресов и примечаний с «Будущей недели».
6. Владелец / расширенный менеджер — позже (отложено).

---

## Экран 1: Менеджер — приём заказа

**Статус:** UI устраивает, логика записи — сломана.

### Должно работать

- День отгрузки, ник/имя, адрес, примечание.
- Корзина: дрессура / жевалки / другое / присыпки / овощи с фракциями и объёмом.
- «Сохранить весь заказ» → столбцы на «Прием заказов» (или «Будущая неделя»).
- Редактирование: перезапись того же клиента (очистка столбца + новая корзина).

### Баги

- Позиции не записываются / пишутся не в ту строку (матчинг без фракции `sub`).
- `fetch` + `no-cors` — ложный «успех».
- Telegram WebApp JS подключён неверно (`telegram.org` вместо `.../telegram-web-app.js`).

---

## Экран 2: Менеджер — записи на неделю

**Статус:** список клиентов читается; отображение единиц и CRM-кнопки — сломаны.

### Должно работать

- Выбор дня → список клиентов с составом, адресом, примечанием.
- Единицы: граммы vs штуки — корректно.
- **Редактировать** → открыть экран 1 с корзиной и сохранить поверх.
- **Удалить** → очистить столбец клиента.
- **Перенести** → на другой день недели / будущую неделю.

### Баги

- Неверные гр/шт (логика категорий в `getClients`).
- delete/move: фронт шлёт **GET/JSONP**, бэкенд обрабатывает только в **doPost**.

---

## Экран 3: Нарезчик (новый)

### Показать

- Выбор дня/даты (как на листе «Нарезка», ячейка A1).
- Только позиции с объёмом **> 0** на этот день.
- Для каждой позиции:
  - сколько нужно **сухого**;
  - сколько нужно **сырого** (через коэффициенты/усушку со «Склада» — колонки уже есть в таблице);
  - чекбокс / статус **«нарезано»** (как флаги на листе «Нарезка»);
  - поле **излишек**, если нарезали больше плана.

### Данные

Источник правды — листы **Нарезка** + **Склад** (+ при необходимости «Прием заказов»). Не дублировать усушку вручную в HTML, если она уже в таблице.

---

## Экран 4: Курьер (новый)

### Показать

- Выбор дня.
- Список клиентов на день: состав заказа, адрес, примечание.
- Чекбоксы **«доставлен»** (синхрон с «Доставки» C2:Q2 или аналог по дню).
- Кнопка: **открыть адреса пачкой в Яндекс.Картах** (построить маршрут / список точек).

---

## Таблица: завершить неделю

### Баг

С «Будущей недели» на «Прием заказов» **не переносятся адреса и примечания** (копируется только до строки 59).

### Ожидание

- Копировать ники + товары + **адрес + примечание** (`C3:Q61`).
- После переноса очищать «Будущую неделю» включая 60–61.
- При очистке Пн–Пт очищать addr/note каждого блока.
- Даты, склад, архив, память — как в текущей `finishFullWeekProduction`, не ломать без ТЗ.
